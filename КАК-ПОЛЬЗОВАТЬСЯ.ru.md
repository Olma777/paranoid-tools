# Paranoid Tools — как пользоваться

Набор из 5 CLI-инструментов приватности для macOS. Каждый делает **одну** вещь
и честно говорит, где гарантий нет (не обещает «100% безвозвратно» там, где SSD
и macOS могут оставить копию).

## Установка (одна команда)

```bash
cd "/Volumes/X10 Pro/projects/paranoid-tools"
bash install.sh
```

Ставит все 5 в `~/.local/bin` (уже в твоём PATH). Повторный запуск = обновление.
Снести всё: `bash install.sh --uninstall`.

Проверка, что встало:

```bash
securetrash version && vaultwatch version && panic version && ghostdraft version
```

## 1. securetrash — стереть файлы и шифрованный сейф

**Когда:** надо удалить чувствительный файл «всерьёз» или хранить секреты в
зашифрованном контейнере.

```bash
securetrash check              # честный вердикт: какие гарантии реальны на твоём диске
securetrash vault create       # создать шифрованный сейф
securetrash vault open         # открыть (смонтировать) сейф
securetrash vault close        # закрыть сейф
securetrash shred ~/secret.pdf # стереть файл (на SSD — через crypto-shred, не перезапись)
```

**Главное:** на SSD «затирание» (`rm -P`) гарантий НЕ даёт. Реальная защита —
держать секреты внутри `vault` (он шифрованный), а «удаление» = уничтожение ключа.
Именно это `check` тебе и объясняет на твоём железе.

## 2. vaultwatch — сторож открытого сейфа

**Когда:** открыл сейф и не хочешь, чтобы он висел открытым и попал в Spotlight /
Time Machine / облако. Обычно работает **сам** через хуки securetrash.

```bash
vaultwatch install-hooks       # один раз: связать с securetrash vault open/close
vaultwatch start --ttl 30m /Volumes/MyVault   # вручную: сторожить + авто-закрыть через 30 мин
vaultwatch stop /Volumes/MyVault              # снять охрану и показать отчёт
```

**Главное:** после `install-hooks` про него можно забыть — `securetrash vault open`
сам поднимет охрану, `close` снимет.

## 3. panic — «всё спрятать, сейчас»

**Когда:** кто-то подошёл / надо мгновенно убрать всё с глаз.

```bash
panic now          # закрыть сейфы, отмонтировать тома, очистить буфер, заблокировать экран
panic now --hard   # то же + убить облачные демоны
```

**Главное:** `panic` ПРЯЧЕТ и БЛОКИРУЕТ, но НЕ уничтожает данные. Удобно повесить
`panic now` на хоткей (через macOS Shortcuts или Raycast → запуск shell-команды).

## 4. ghostdraft — черновик без следов на диске

**Когда:** надо набросать секретный текст (пароль, seed, заметку), который не должен
осесть на диске.

```bash
ghostdraft new              # открыть эфемерный черновик; на выходе — стереть + чистка истории редактора
ghostdraft new --clipboard  # то же + положить результат в буфер (по умолчанию ВЫКЛ)
pbpaste | ghostdraft pipe   # показать содержимое буфера в терминале, НЕ записывая на диск
```

**Главное:** ghostdraft не обещает «ноль следов» там, где их оставляет сама ОС
(swap, scrollback терминала) — он это честно перечисляет.

## 5. seedsplit — разрезать секрет на доли (схема Шамира)

**Когда:** есть seed-фраза / мастер-пароль / приватный ключ, и не хочешь, чтобы один
носитель (бумажка, флешка, бэкап) был единой точкой провала. Режешь на N частей так,
что любые **T** из них восстанавливают секрет, а T−1 не дают о нём **ничего**.

```bash
# Разбить seed на 5 долей, порог 3 (любые 3 из 5 восстановят). Секрет — через stdin:
printf '%s' "legal winner thank year wave ..." | seedsplit split -n 5 -t 3
# → 5 строк вида SSS1-3-1-<hex>-<chk>. Разнеси их по разным местам.

# Собрать обратно из любых 3 долей (по строке на долю):
seedsplit combine < мои-3-доли.txt
# или: pbpaste | seedsplit combine
```

**Главное:**
- Секрет подаётся через **stdin или `--file`**, не аргументом (аргумент виден в `ps`).
- Каждая доля несёт контрольную сумму — **опечатку при наборе доли поймает**.
- Если смешать доли от РАЗНЫХ секретов или повредить — combine **честно откажет**,
  а не вернёт мусор (внутри секрета зашита проверка целостности).
- ⚠️ Пока **нет** совместимости со SLIP-39 / аппаратными кошельками — это собственный
  формат. Доли безопасны ровно настолько, насколько надёжно ты их **хранишь и разносишь**.

## Типичный сценарий (связка)

```bash
securetrash vault open                  # открыл сейф (vaultwatch сам встал на охрану)
ghostdraft new                          # набросал секрет, он лёг внутрь сейфа, следов снаружи нет
# ...работаешь...
panic now                               # кто-то подошёл — мгновенно всё закрыл и заблокировал экран
```

## Подсказки

- Любой тул без аргументов или с непонятной командой печатает свою справку.
- `securetrash check` — первое, что стоит запустить: он объяснит, что на твоём
  железе реально защищает, а что нет.
