# Paranoid Tools — инструкция по запуску

Пять CLI-инструментов приватности для macOS. Каждый делает **одну** вещь и честно
говорит, где гарантий нет — не обещает «100% безвозвратно» там, где SSD и macOS
могут оставить копию.

## Установка (одна команда)

```bash
git clone https://github.com/Di-kairos/paranoid-tools
cd paranoid-tools
bash install.sh
```

Ставит все 5 инструментов и лаунчер `paranoid` в `~/.local/bin` (уже в твоём PATH).
Повторный запуск = обновление. Снести всё: `bash install.sh --uninstall`.

На свежем клоне каждый инструмент тянется из подписанного релиза с проверкой подписи
и контрольной суммы — ничего не выполняется, пока не проверено.

## С чего начать — запусти `paranoid`

Главная точка входа. Одна команда открывает дашборд со статусом и меню поверх всех
пяти инструментов:

```bash
paranoid
```

Сверху — живой статус (обновляется при каждом возврате в меню):

```
Vault:       ● open (/Volumes/SecretVault)   ⚠ at risk while open
FileVault:   ● on
vaultwatch:  ● active
```

- **Vault** — открыт сейф или закрыт. Открытый помечается «at risk»: пока он
  смонтирован, данные доступны любому, кто за твоим Mac.
- **FileVault** — включено ли системное шифрование диска.
- **vaultwatch** — сторожит ли сейчас открытый сейф (`active`) или сессий нет (`idle`).

Ниже — меню (выбор по цифре):

```
1) Status        — полная read-only проверка (что реально защищает на твоём железе)
2) 🔒 PANIC NOW  — спрятать и запереть всё (с подтверждением)
3) Vault         — создать / открыть / закрыть сейф
4) Split         — разрезать секрет на доли (seedsplit)
5) Combine       — собрать секрет из долей (seedsplit)
6) Ghostdraft    — черновик без следов: 1) new  2) pipe  3) new + копия в буфер
7) Watch vault   — взять открытый сейф под охрану + TTL (vaultwatch)
0) Quit
```

Лаунчер секретов не хранит и своей крипты не добавляет: он запускает те же пять
подписанных CLI и показывает их вывод как есть — со всеми оговорками и вердиктами.
Всё, что делает меню, доступно и напрямую — команды ниже.

## Инструменты по отдельности

### 1. securetrash — стереть файлы и шифрованный сейф

**Когда:** надо удалить чувствительный файл всерьёз или хранить секреты в зашифрованном
контейнере.

```bash
securetrash check              # честный вердикт: какие гарантии реальны на твоём диске
securetrash vault create       # создать шифрованный сейф
securetrash vault open         # открыть (смонтировать) сейф
securetrash vault close        # закрыть сейф
securetrash shred ~/secret.pdf # стереть файл (на SSD — crypto-shred, не перезапись)
```

**Главное:** на SSD затирание (`rm -P`) гарантий не даёт. Реальная защита — держать
секреты внутри `vault` (он шифрованный), а «удаление» = уничтожение ключа. Именно это
`check` объясняет на твоём железе.

**Открытый сейф в Finder.** После `vault open` том виден как обычный диск: в боковой
панели Finder («Места») и на рабочем столе, окно открывается само. Закрыл окно — том
никуда не делся: открой его снова из боковой панели, или командой `open /Volumes/SecretVault`,
или в Finder через `Cmd+Shift+G` → `/Volumes/SecretVault`. «Извлечь» том из Finder
размонтирует его — это тот же `vault close`, секреты снова зашифрованы на диске. Если
нужна максимальная приватность (том вне боковой панели и рабочего стола) — открывай с
`ST_VAULT_HIDDEN=1 securetrash vault open`; тогда вернуть окно можно только командой `open`.

### 2. vaultwatch — сторож открытого сейфа

**Когда:** открыл сейф и не хочешь, чтобы он висел открытым и попал в Spotlight,
Time Machine или облако. Обычно работает сам через хуки securetrash.

```bash
vaultwatch install-hooks                       # один раз: связать с vault open/close
vaultwatch start --ttl 30m /Volumes/MyVault    # вручную: охрана + авто-закрытие через 30 мин
vaultwatch stop /Volumes/MyVault               # снять охрану и показать отчёт
vaultwatch status                              # какие сейфы под охраной сейчас
```

**Главное:** после `install-hooks` про него можно забыть — `securetrash vault open`
сам поднимет охрану, `close` снимет.

### 3. panic — спрятать всё, сейчас

**Когда:** кто-то подошёл, надо мгновенно убрать всё с глаз.

```bash
panic now            # закрыть сейфы, отмонтировать тома, очистить буфер, заблокировать экран
panic now --hard     # то же + убить облачные демоны
panic hotkey install # повесить на глобальный хоткей (cmd + alt - p), см. README panic
```

**Главное:** `panic` прячет и блокирует, но данные не уничтожает. Для настоящей скорости
повесь его на глобальный хоткей через `panic hotkey`.

### 4. ghostdraft — черновик без следов на диске

**Когда:** надо набросать секретный текст (пароль, seed, заметку), который не должен
осесть на диске.

```bash
ghostdraft new              # эфемерный черновик; на выходе — стереть + чистка следов редактора
ghostdraft new --clipboard  # то же + на выходе положить результат в буфер обмена (с подтверждением)
pbpaste | ghostdraft pipe   # показать буфер в терминале, не записывая на диск
```

**Сначала пойми, что тебе нужно — это две разные задачи:**

- **Написал секрет → отдал/вставил один раз → пусть исчезнет.** Это `ghostdraft new`.
  Файл живёт внутри открытого `vault` (или на RAM-диске, если сейфа нет), а на выходе из
  редактора — стирается. Записывать сюда ключи «на память» бессмысленно: после выхода их
  нет. Чтобы успеть забрать результат — см. `--clipboard` ниже.
- **Держать ключи и копировать их в приложения снова и снова.** Это НЕ `ghostdraft new`
  (он шредит на выходе). Заведи обычный файл **внутри открытого сейфа** —
  `/Volumes/SecretVault/keys.txt` — редактируй и копируй когда нужно, а `securetrash vault
  close` снова зашифрует его на диске. Это твоё постоянное зашифрованное хранилище.

**Скопировать секрет один раз (через лаунчер):**

1. `paranoid` → `6` (Ghostdraft) → `3` (new + копия в буфер).
2. Откроется редактор (`vim` с мягким переносом — длинный ключ не уезжает за край и не
   рвётся). Впиши секрет, сохрани и выйди: `:wq`.
3. ghostdraft спросит подтверждение на копирование в буфер — ответь `yes`.
4. Вставь секрет в нужное приложение (`Cmd+V`).
5. Через ~20 секунд буфер очистится сам — но только если ты за это время его не перезаписал
   (скопировал что-то другое). Сам черновик к этому моменту уже стёрт.

То же напрямую, без меню: `ghostdraft new --clipboard`.

**Главное:** ghostdraft не обещает «ноль следов» там, где их оставляет сама ОС (swap,
scrollback терминала, у `vim` — `~/.viminfo`) — он это честно перечисляет на выходе.
Дефолтный редактор `vim -i NONE` как раз отключает `~/.viminfo`. Буфер обмена опасен сам
по себе (менеджеры буфера, Universal Clipboard в iCloud) — потому копирование явное, с
подтверждением, и с авто-очисткой.

### 5. seedsplit — разрезать секрет на доли (схема Шамира)

**Когда:** есть seed-фраза, мастер-пароль или приватный ключ, и не хочешь, чтобы один
носитель (бумажка, флешка, бэкап) был единой точкой провала. Режешь на N частей так, что
любые **T** из них восстанавливают секрет, а T−1 не дают о нём **ничего**.

```bash
# Разбить seed на 5 долей, порог 3 (любые 3 из 5 восстановят). Секрет — через stdin:
printf '%s' "legal winner thank year wave ..." | seedsplit split -n 5 -t 3
# → 5 строк вида SSS1-3-1-<hex>-<chk>. Разнеси их по разным местам.

# Собрать обратно из любых 3 долей (по строке на долю):
seedsplit combine < мои-3-доли.txt
# или: pbpaste | seedsplit combine
```

**Главное:**
- Секрет подаётся через **stdin или `--file`**, не аргументом (аргумент виден в `ps`).
- Каждая доля несёт контрольную сумму — опечатку при наборе доли поймает.
- Доли от разных секретов или повреждённые — `combine` честно откажет, а не вернёт мусор
  (внутри зашита проверка целостности).
- ⚠️ Совместимости со SLIP-39 и аппаратными кошельками пока нет — это собственный формат.
  Доли безопасны ровно настолько, насколько надёжно ты их хранишь и разносишь.

## Типичный сценарий

```bash
securetrash vault open    # открыл сейф — vaultwatch сам встал на охрану
ghostdraft new            # набросал секрет, он лёг внутрь сейфа, следов снаружи нет
# ...работаешь...
panic now                 # кто-то подошёл — мгновенно всё закрыл и заблокировал экран
```

## Подсказки

- Любой инструмент без аргументов или с непонятной командой печатает свою справку.
- `securetrash check` стоит запустить первым: он объяснит, что на твоём железе реально
  защищает, а что нет.
- Не помнишь команды — открой `paranoid` и работай через меню.
